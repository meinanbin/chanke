## 1. 注册微信小程序账号

### 1.1 进入注册页面

1. 打开浏览器，访问 **https://mp.weixin.qq.com/**
2. 点击页面右上角的 **「立即注册」** 按钮

### 1.2 选择账号类型

1. 在账号类型选择页面，点击 **「小程序」** 下方的 **「前往注册」**
2. 注意：不要选择"公众号"或"企业微信"，必须选择"小程序"

### 1.3 填写注册信息

1. **邮箱**：
   - 填写一个 **未曾注册过微信公众号/小程序/个人微信号** 的邮箱
   - 建议使用企业邮箱或新建邮箱，不要使用个人微信绑定的邮箱
   - 此邮箱将作为登录账号
2. **密码**：设置 8~16 位密码（需包含字母和数字）
3. **确认密码**：再次输入密码
4. 勾选 **「我同意」** 服务协议
5. 点击 **「注册」**

### 1.4 邮箱激活

1. 登录你填写的邮箱，查找来自微信团队的激活邮件
2. 点击邮件中的 **激活链接**，完成邮箱验证
3. 激活后页面自动跳转到信息登记步骤

### 1.5 信息登记（主体类型选择）

根据你的实际情况选择主体类型：

| 主体类型 | 适用场景 | 是否需要认证 |
|---------|---------|------------|
| 企业 | 公司、企业 | 是（需营业执照） |
| 个体工商户 | 个体经营者 | 是（需营业执照） |
| 政府/媒体/其他组织 | 机关单位、媒体等 | 是（需组织机构代码） |
| 个人 | 个人开发者 | 否（功能受限） |

**以企业主体为例：**

1. 主体类型选择 **「企业」**
2. 填写 **企业名称**（与营业执照一致）
3. 填写 **营业执照注册号**（统一社会信用代码）
4. 上传 **营业执照照片**（扫描件或拍照，需清晰）
5. 填写 **管理员姓名**（法定代表人或经办人）
6. 填写 **管理员身份证号**
7. 上传 **管理员身份证正反面照片**
8. 填写 **管理员手机号**
9. 使用管理员微信 **扫码验证**（扫码后微信会确认身份）
10. 点击 **「继续」** 提交

> 提交后微信团队会在 **1~3 个工作日** 内完成审核，审核结果通过短信和微信通知。

## 2. 完成小程序认证

### 2.1 为什么需要认证

- 未认证的小程序功能受限（无法使用微信支付、无法获取用户手机号等）
- 已认证的小程序才能提交代码审核并发布正式版
- **个人主体无需认证**，但功能较少

### 2.2 认证操作步骤

1. 登录 **https://mp.weixin.qq.com/**
2. 使用注册时的邮箱和密码登录
3. 左侧菜单 → **设置** → **基本设置**
4. 找到 **「微信认证」** 区域，点击 **「详情」**
5. 点击 **「办理」** 开始认证流程
6. 认证方式选择 **「法定代表人验证」**（推荐）或 **「对公账户打款验证」**
7. 支付认证费用 **300 元/年**（微信支付扫码付款）
8. 填写认证联系人信息并上传相关材料
9. 等待第三方审核机构联系（通常 **1~3 个工作日**）
10. 审核通过后，小程序状态变为 **「已认证」**

> 法定代表人验证方式最快，通常当天即可完成。

## 3. 获取关键信息（AppID / AppSecret）

### 3.1 获取 AppID

AppID 是小程序的唯一标识，开发者工具必须填写才能上传代码。

1. 登录 **https://mp.weixin.qq.com/**
2. 左侧菜单 → **开发** → **开发管理** → **开发设置**
3. 在页面顶部可以看到 **AppID（小程序ID）**，形如 `wx1234567890abcdef`
4. **复制并保存** 这个 AppID

### 3.2 获取 AppSecret

AppSecret 是小程序的密钥，用于服务端接口调用（如获取 access_token）。

1. 在同一页面找到 **AppSecret（小程序密钥）**
2. 点击 **「生成」** 或 **「重置」**
3. 用管理员微信扫码确认
4. 系统会显示 AppSecret，**立即复制保存**（关闭后无法再次查看，只能重置）
5. 将 AppID 和 AppSecret 保存在安全的地方

### 3.3 配置 IP 白名单

1. 在开发设置页面找到 **「IP白名单」**
2. 点击 **「修改」**
3. 填写你的后端服务器 IP 地址（每行一个）
4. 只有白名单内的 IP 才能使用 AppSecret 获取 access_token

## 4. 配置服务器域名（必须）

> **⚠️ 本项目使用后端 API（wx.request），正式版必须配置合法服务器域名。**
> 开发阶段可在开发者工具中临时跳过域名校验，但正式版上线前必须完成域名配置。

### 4.1 前置条件：域名备案 + SSL 证书

#### 4.1.1 购买域名

1. 在域名注册商（阿里云、腾讯云、GoDaddy 等）购买域名
2. 建议选择 `.com` 或 `.cn` 域名
3. 示例：`obstetric-api.yourdomain.com`

#### 4.1.2 ICP 备案

> 域名必须通过工信部 ICP 备案才能在微信小程序中使用。

1. 在云服务商（阿里云、腾讯云等）的备案系统中提交备案申请
2. 填写网站信息：
   - **网站名称**：产科检查管理系统
   - **域名**：你购买的域名
   - **服务器 IP**：后端服务器公网 IP
3. 上传营业执照、身份证等材料
4. 备案审核通常需要 **7~20 个工作日**
5. 备案成功后，云服务商会发送备案号

#### 4.1.3 配置 SSL 证书（HTTPS）

> 微信小程序要求所有网络请求必须使用 HTTPS。

**方式一：免费证书（推荐开发/测试）**

1. 在 [Let's Encrypt](https://letsencrypt.org/) 申请免费 SSL 证书
2. 使用 certbot 工具自动安装：
   ```bash
   # 安装 certbot
   apt-get install certbot   # Ubuntu/Debian
   yum install certbot        # CentOS

   # 申请证书（需先配置域名 DNS 解析到服务器 IP）
   certbot certonly --standalone -d obstetric-api.yourdomain.com

   # 证书会保存在 /etc/letsencrypt/live/obstetric-api.yourdomain.com/
   # fullchain.pem = 证书链
   # privkey.pem = 私钥
   ```

**方式二：付费证书（推荐生产环境）**

1. 在云服务商购买 SSL 证书（约 100~1000 元/年）
2. 提交域名验证（DNS 解析验证或文件验证）
3. 下载证书文件（Nginx 格式：`.pem` + `.key`）

### 4.2 配置域名 DNS 解析

1. 登录域名管理面板（云服务商 DNS 管理）
2. 添加 A 记录：
   - **记录类型**：A
   - **主机记录**：`obstetric-api`（或你选择的子域名前缀）
   - **记录值**：后端服务器公网 IP 地址
   - **TTL**：600（10 分钟）
3. 等待 DNS 解析生效（通常几分钟到几小时）
4. 验证解析：在终端执行 `ping obstetric-api.yourdomain.com`，应返回服务器 IP

### 4.3 在微信公众平台配置服务器域名

1. 登录 **https://mp.weixin.qq.com/**
2. 左侧菜单 → **开发** → **开发管理** → **开发设置**
3. 向下滚动找到 **「服务器域名」** 区域，点击 **「修改」**
4. 用管理员微信扫码确认
5. 在 **「request 合法域名」** 中填写你的后端 API 地址：
   - 格式必须是 HTTPS，如：`https://obstetric-api.yourdomain.com`
   - 不含端口、不含路径
   - **一行一个域名，最多 200 个**
6. 其他域名类型按需填写：

| 域名类型 | 本项目是否需要 | 说明 |
|---------|-------------|------|
| request 合法域名 | ✅ 必须 | wx.request 调用后端 API |
| socket 合法域名 | ❌ 不需要 | 本项目无 WebSocket |
| uploadFile 合法域名 | ❌ 不需要 | 本项目无文件上传 |
| downloadFile 合法域名 | ❌ 不需要 | 本项目无文件下载 |
| udp 合法域名 | ❌ 不需要 | 本项目无 UDP |

7. 点击 **「保存」**

> **⚠️ 重要**：域名配置后，修改次数有月度限制（每月最多修改 5 次），请谨慎填写。

### 4.4 开发阶段跳过域名校验

在开发阶段，如果尚未配置域名或域名未备案，可以在开发者工具中临时跳过校验：

1. 打开微信开发者工具
2. 点击右上角 **「详情」** → **「本地设置」**
3. 勾选 **「不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书」**
4. 注意：此设置仅在开发环境生效，**正式版必须配置合法域名**

## 5. 部署后端服务器

> 本项目的后端服务器为 Node.js Express + SQLite，需部署到公网云服务器才能让小程序正式版访问。

### 5.1 选择云服务器

| 云服务商 | 推荐配置 | 月费用参考 | 说明 |
|---------|---------|-----------|------|
| 阿里云 ECS | 1核2G / 40G云盘 | 约 40~60 元 | 国内最常用，备案方便 |
| 腾讯云 CVM | 1核2G / 50G云盘 | 约 35~50 元 | 微信生态友好 |
| 华为云 ECS | 1核2G / 40G云盘 | 约 30~50 元 | 适合政企场景 |

> 本项目数据量小（SQLite），最低配置即可满足需求。如数据量增长，可考虑升级。

### 5.2 云服务器基础环境配置

#### 5.2.1 连接服务器

1. 使用 SSH 工具连接服务器：
   ```bash
   ssh root@你的服务器IP
   ```
2. 或使用云服务商提供的 Web 终端

#### 5.2.2 安装 Node.js

```bash
# Ubuntu/Debian
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt-get install -y nodejs

# CentOS
curl -fsSL https://rpm.nodesource.com/setup_20.x | bash -
yum install -y nodejs

# 验证安装
node -v   # 应显示 v20.x.x
npm -v    # 应显示 10.x.x
```

#### 5.2.3 安装 Nginx（反向代理 + HTTPS）

```bash
# Ubuntu/Debian
apt-get install -y nginx

# CentOS
yum install -y nginx

# 启动 Nginx
systemctl start nginx
systemctl enable nginx   # 开机自启

# 验证
curl http://localhost   # 应显示 Nginx 默认页面
```

#### 5.2.4 防火墙配置

```bash
# 开放必要端口
# Ubuntu (ufw)
ufw allow 22    # SSH
ufw allow 80    # HTTP
ufw allow 443   # HTTPS
ufw enable

# CentOS (firewalld)
firewall-cmd --permanent --add-service=ssh
firewall-cmd --permanent --add-service=http
firewall-cmd --permanent --add-service=https
firewall-cmd --reload
```

### 5.3 上传后端代码到服务器

#### 方式一：Git 克隆（推荐）

```bash
# 在服务器上安装 Git（如未安装）
apt-get install -y git   # Ubuntu
yum install -y git        # CentOS

# 克隆代码（假设代码已推送到 Git 仓库）
cd /opt
git clone https://github.com/your-org/obstetric-exam-miniapp.git

# 进入后端目录
cd /opt/obstetric-server
```

#### 方式二：手动上传

1. 在本地打包 server 目录：

   **方式 A — PowerShell（推荐，Windows 自带）**：
   ```powershell
   # 打开 PowerShell，执行：
   Compress-Archive -Path "C:\Users\Administrator\Documents\2026-06-27-08-06-17\server\*" -DestinationPath "C:\Users\Administrator\Documents\2026-06-27-08-06-17\server.zip" -Force
   ```

   **方式 B — Git Bash（如已安装 Git）**：
   ```bash
   # 在 Git Bash 中执行（注意用正斜杠路径）：
   cd /c/Users/Administrator/Documents/2026-06-27-08-06-17
   tar -czf server.tar.gz server/
   ```

   > **⚠️ 为何不要在 Windows CMD 中直接使用 `tar` 命令？**
   >
   > Windows 10/11 自带的 `tar.exe` 通常位于 `C:\Program Files (x86)\...` 或 `C:\Windows\System32\tar.exe`。
   > 当通过 CMD 调用时，如果系统 `PATH` 环境变量中存在含空格的路径（如 `C:\Program Files (x86)\...`），
   > `tar` 会将路径中的 `Program Files (x86)` 拆分，误把 `ogram Files (x86)` 当作目录参数去访问，
   > 从而报错 `tar: ogram Files (x86): Couldn't visit directory: No such file or directory`。
   >
   > **解决方案**：使用 PowerShell 的 `Compress-Archive`，或使用 Git Bash 中的 tar。

2. 上传到服务器：

   > **⚠️ 注意**：`scp` 命令中的本地文件路径必须是**完整路径**或确保命令行当前目录正确。
   > 如果报错 `stat local "server.zip": No such file or directory`，说明当前目录不是文件所在目录，请使用完整路径。

   ```bash
   # PowerShell 上传 .zip（使用完整路径，避免找不到文件）：
   scp "C:\Users\Administrator\Documents\2026-06-27-08-06-17\server.zip" root@你的服务器IP:/opt/

   # 或 Git Bash 上传 .tar.gz（用正斜杠路径）：
   scp /c/Users/Administrator/Documents/2026-06-27-08-06-17/server.tar.gz root@你的服务器IP:/opt/
   ```
3. 在服务器上解压：
   ```bash
   # 如果上传的是 .zip：
   cd /opt
   unzip server.zip -d obstetric-server
   cd obstetric-server

   # 如果上传的是 .tar.gz：
   cd /opt
   tar -xzf server.tar.gz
   cd server
   ```

### 5.5 配置 Nginx 反向代理

创建 Nginx 配置文件：

```bash
# 创建配置文件
nano /etc/nginx/sites-available/obstetric-api
```

#### 方式一：HTTP 临时配置（尚未有域名/SSL 证书时使用）

> 如果你还没有购买域名、未完成备案或尚未申请 SSL 证书，先用这个简化配置，通过 IP 直接访问。
> **注意：微信正式版必须使用 HTTPS，此配置仅供开发测试使用。**

```nginx
# HTTP 临时配置（无 SSL，通过 IP 访问）
server {
    listen 80;
    server_name 139.199.71.127;  # 替换为你的服务器公网 IP

    # 反向代理到后端 Node.js 服务（本地端口 3000）
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # 超时配置
        proxy_connect_timeout 60s;
        proxy_read_timeout 120s;
        proxy_send_timeout 60s;
    }

    # 健康检查不缓存
    location /health {
        proxy_pass http://127.0.0.1:3000;
        add_header Cache-Control no-store;
    }
}
```

写入后保存（nano 中按 `Ctrl+O` 回车保存，`Ctrl+X` 退出）。

验证：`curl http://139.199.71.127/health`（替换为你的 IP）应返回 `{"status":"ok"}`。

#### 方式二：HTTPS 完整配置（已有域名 + SSL 证书时使用）

> 域名备案通过、SSL 证书申请完成后，替换为此配置。

```nginx
# HTTP → HTTPS 重定向
server {
    listen 80;
    server_name obstetric-api.yourdomain.com;  # 替换为你的真实域名
    return 301 https://$server_name$request_uri;
}

# HTTPS 主配置
server {
    listen 443 ssl;
    server_name obstetric-api.yourdomain.com;  # 替换为你的真实域名

    # SSL 证书路径（替换为你的真实域名对应的证书路径）
    ssl_certificate     /etc/letsencrypt/live/obstetric-api.yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/obstetric-api.yourdomain.com/privkey.pem;

    # SSL 安全配置
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;

    # 反向代理到后端 Node.js 服务（本地端口 3000）
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # 超时配置
        proxy_connect_timeout 60s;
        proxy_read_timeout 120s;
        proxy_send_timeout 60s;
    }

    # 健康检查不缓存
    location /health {
        proxy_pass http://127.0.0.1:3000;
        add_header Cache-Control no-store;
    }
}
```

> **⚠️ 关键提醒**：上面的 `obstetric-api.yourdomain.com` 是示例占位符，必须替换为你自己的真实域名。如果你还没有域名和 SSL 证书，请先使用「方式一」HTTP 临时配置。

启用配置并重启 Nginx：

```bash
# 启用站点配置
ln -s /etc/nginx/sites-available/obstetric-api /etc/nginx/sites-enabled/

# 测试配置
nginx -t

# 重载 Nginx
systemctl reload nginx
```

> **CentOS 用户**：Nginx 默认使用 `/etc/nginx/conf.d/` 目录，将配置文件放到该目录即可，无需 `sites-available/sites-enabled` 链接。

### 5.6 使用 PM2 管理后端进程（推荐）

PM2 是 Node.js 进程管理器，可自动重启、日志管理、开机自启。

```bash
# 安装 PM2

# 查看运行状态

# 查看日志

# 设置开机自启

pm2 save
```

PM2 常用命令：

| 命令 | 说明 |
|------|------|

| `pm2 delete obstetric-api` | 删除进程 |

#!/bin/bash
BACKUP_DIR="/opt/backups/obstetric"

### 15.5 后端运维

| 操作 | 命令 |
|------|------|
| 查看服务状态 | `pm2 status` |
| 重启服务 | `pm2 restart obstetric-api` |
| 查看实时日志 | `pm2 logs obstetric-api` |
| 查看数据库大小 | `ls -lh /opt/obstetric-server/obstetric.db` |
| 手动备份数据库 | `cp /opt/.../server/obstetric.db /opt/backups/obstetric/db_$(date +%Y%m%d).db` |
| SSL 证书续期 | `certbot renew`（Let's Encrypt 每 90 天需续期） |
| Nginx 配置修改后 | `nginx -t && systemctl reload nginx` |
| 启动 Nginx | `systemctl start nginx` |
| 停止 Nginx | `systemctl stop nginx` |
| 重启 Nginx | `systemctl restart nginx` |
| 查看 Nginx 运行状态 | `systemctl status nginx` |
| 设置 Nginx 开机自启 | `systemctl enable nginx` |
| 取消 Nginx 开机自启 | `systemctl disable nginx` |
| 平滑重载配置（不中断连接） | `systemctl reload nginx` |
| 强制停止 Nginx（立即断开） | `nginx -s stop` |
| 优雅退出 Nginx（处理完请求后退出） | `nginx -s quit` |

> **停止 vs 重启 Nginx 说明**：
> - **`systemctl stop`**：完全停止 Nginx 服务，所有连接立即断开，用户无法访问
> - **`systemctl reload`**：平滑重载配置文件，不断开现有连接，推荐修改配置后使用
> - **`nginx -s quit`**：优雅退出，等待当前请求处理完毕后关闭，适合维护窗口
> - **`nginx -s stop`**：强制快速停止，不等待请求完成
> - **停止 Nginx 后小程序将无法访问后端 API**，请谨慎操作

## 16. 常见问题与排查

### 16.1 上传失败

| 问题 | 解决方案 |
|------|---------|
| 提示「AppID不合法」 | 检查 project.config.json 中 appid 是否为真实 AppID |
| 提示「包体积过大」 | 主包不超过 2MB，检查是否有大图片/未使用文件 |
| 提示「没有权限」 | 确认当前微信号是小程序管理员或开发者 |
| 提示「网络错误」 | 检查网络连接，关闭代理/VPN 后重试 |

### 16.2 审核被拒

| 问题 | 解决方案 |
|------|---------|
| 「功能过于简单」 | 补充更多功能页面，确保核心流程完整 |
| 「页面无法操作」 | 确认后端 API 运行正常，数据可正常加载 |
| 「测试账号无效」 | 确保提供的账号密码可正常登录 |
| 「缺少隐私协议」 | 填写用户隐私保护指引 |
| 「域名未配置」 | 在公众平台配置 request 合法域名 |

### 16.3 API 请求失败

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 小程序端提示「网络请求失败」 | 后端未启动 | 启动后端：`node server.js` 或 `pm2 restart obstetric-api` |
| 小程序端提示「HTTP 错误：404」 | API 路径错误 | 检查 `api.js` 中 `API_BASE` 和接口路径 |
| 小程序端提示「HTTP 错误：500」 | 后端代码错误 | 查看后端日志：`pm2 logs obstetric-api` |
| 正式版 API 请求全部失败 | 域名未配置或 HTTPS 异常 | 检查公众平台服务器域名配置 + SSL 证书 |
| 开发版 API 请求失败 | 未勾选「不校验合法域名」 | 开发者工具 → 详情 → 本地设置勾选 |

### 16.4 后端服务器问题

| 问题 | 解决方案 |
|------|---------|
| 服务器无法启动 | 检查 Node.js 版本（需 v16+），检查依赖是否安装（`npm install`） |
| better-sqlite3 安装失败 | 安装构建工具：`apt-get install build-essential python3` |
| 数据库文件损坏 | 使用备份恢复：`cp /opt/backups/obstetric/db_最新日期.db obstetric.db` |
| PM2 进程异常退出 | 查看日志：`pm2 logs obstetric-api`，排查错误后重启 |
| HTTPS 证书过期 | 执行续期：`certbot renew && systemctl reload nginx` |
| Nginx 502 Bad Gateway | 后端进程未运行，启动：`pm2 restart obstetric-api` |
| 端口 3000 被占用 | 查看占用进程：`lsof -i :3000`，或修改 server.js 端口 |

### 16.5 体验版/正式版数据异常

| 问题 | 解决方案 |
|------|---------|
| 打开小程序后数据为空 | 后端未运行或 API 地址错误，检查 `api.js` 和后端服务状态 |
| 不同用户看到不同数据 | 检查后端数据库是否共用，确认所有用户连接同一 API 地址 |
| 操作后数据未更新 | 检查 API 写入请求是否成功（Network 面板），查看后端日志 |
| 数据丢失 | 检查后端数据库文件是否正常，如有损坏从备份恢复 |

### 16.6 开发环境调试

| 问题 | 解决方案 |
|------|---------|
| 本地编译后 API 报错 | 确认后端服务器已在本地启动（`node server.js`） |
| `api.js` 指向 localhost 但真机无法访问 | 真机需与电脑在同一网络，或改为内网 IP（如 `http://192.168.1.x:3000/api`） |
| `better-sqlite3` 编译报错 | Windows 需安装 Visual Studio Build Tools；Linux 需安装 `build-essential` |
